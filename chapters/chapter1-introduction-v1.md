# Chapter 1: Introduction

## 1.1 研究背景

隨著大數據與人工智慧技術的快速發展,機器學習模型訓練對數據規模和多樣性的需求日益增加。然而,醫療診斷、金融服務、智慧城市等領域涉及大量敏感個人數據,受到歐盟通用數據保護規則(GDPR)和中國個人信息保護法等嚴格法規約束,禁止原始數據跨機構共享。聯邦學習(Federated Learning, FL)允許各參與方在本地訓練模型而無需共享原始數據,僅交換模型更新參數,提供了隱私保護的分散式機器學習範式。

傳統聯邦學習採用中央化聚合器架構,由單一參數服務器負責收集、聚合和分發全局模型。這種架構存在三大根本性缺陷:首先,**單點故障(SPOF)問題**嚴重威脅系統可用性,聚合器故障導致整個訓練中斷;其次,**性能瓶頸**隨參與節點數線性增長,實證研究顯示參與節點超過1000時單點聚合延遲可達數十秒;第三,**安全性風險**要求參與者完全信任中央服務器,缺乏對惡意聚合器的防禦能力,單個惡意聚合器即可破壞全局模型。

區塊鏈技術提供去中心化信任層和不可篡改記錄機制,為解決聯邦學習的信任問題提供了新途徑。區塊鏈聯邦學習(Blockchain-based Federated Learning, BFL)將模型更新記錄上鏈,透過共識機制驗證聚合結果正確性。多聚合器架構進一步分散計算負載,每個聚合器僅負責部分訓練輪次,提升容錯能力與可擴展性。然而,多聚合器協調需要高效且安全的共識機制,這構成了當前研究的核心挑戰。

## 1.2 研究問題與動機

多聚合器區塊鏈聯邦學習架構需要共識機制協調驗證聚合結果,確保在惡意節點存在時仍能維持模型訓練的正確性。當前主流共識機制面臨效率與安全性的根本性矛盾。

**實用拜占庭容錯(Practical Byzantine Fault Tolerance, PBFT)協議**提供嚴格的拜占庭容錯保證,可容忍最多f < n/3個惡意節點。然而,PBFT存在嚴重的可擴展性瓶頸:其通訊複雜度為O(n²),意味著每輪共識需要的消息數量隨節點數平方增長。實證研究顯示,當驗證節點數n > 100時,PBFT協議幾乎不可行,每輪共識延遲可達數十秒。

為解決PBFT的效率問題,現有研究提出了多種優化方案,但仍存在顯著局限:

- **FLCoin系統**採用滑動窗口委員會機制將複雜度從O(n²)降至O(c),但**委員會規模固定**,無法根據實時威脅等級動態調整安全強度,面臨「過度驗證浪費資源」與「驗證不足存在風險」的兩難困境。

- **Optimistic Machine Learning (opML)**採用樂觀執行假設大多數參與者誠實,正常情況下無需驗證實現極高效率,但**完全缺乏拜占庭容錯保證**,僅依賴經濟激勵阻止攻擊,若攻擊收益超過質押損失則安全模型失效。

- **BlockDFL系統**結合PBFT投票與異常檢測實現40%惡意容錯,但**每輪都執行PBFT驗證**,即使威脅為零仍需承擔O(V²)驗證開銷,缺乏樂觀執行模式。

- **EPP-BCFL系統**結合權益證明(PoS)與拜占庭容錯(BFT)形成混合共識,但**固定執行兩階段流程**,非威脅感知型動態切換,無法根據實時安全狀況調整驗證強度。

現有方案的共同缺陷在於採用**靜態共識策略**:要麼始終執行高成本驗證犧牲效率,要麼完全依賴樂觀假設犧牲安全性,缺乏根據威脅等級動態調整的自適應機制。這引出本研究的核心問題:

**如何在多聚合器區塊鏈聯邦學習中實現威脅感知的動態安全強度調整,在正常情況保持接近O(R/N)的高效率,檢測到威脅時自動切換至f < n/3的拜占庭容錯保證,同時避免O(n²)通訊複雜度瓶頸?**

## 1.3 研究貢獻

本研究提出一個混合Optimistic-PBFT區塊鏈聯邦學習框架,通過威脅感知的動態共識切換機制實現安全性與效率的最優平衡。主要貢獻包括五個方面:

**1. 混合Optimistic-PBFT共識機制**

提出動態切換的混合共識協議,正常情況採用樂觀執行模式實現O(R/N)計算複雜度,檢測到威脅時自動切換至PBFT驗證模式實現O(R×V)複雜度。相較於純PBFT的O(R×V²)複雜度,效率提升達N×V/f倍,其中N為聚合器總數、V為驗證者數量、f為最大容忍惡意節點數。理論分析證明在挑戰率p < 10%時,平均複雜度接近樂觀模式的O(R/N),同時保持PBFT等級的f < n/3拜占庭容錯保證。

**2. 威脅感知的挑戰機制設計**

設計多層威脅檢測框架結合挑戰驗證機制,允許驗證者在挑戰期間質疑可疑聚合結果。威脅檢測包含統計異常檢測、行為模式分析、交叉驗證三層機制,挑戰成功時自動觸發PBFT共識並回滾至安全狀態。相較於opML僅依賴欺詐證明的經濟安全,本框架提供密碼學與共識層的雙重安全保證,即使質押機制失效仍可通過PBFT驗證確保模型正確性。

**3. 多聚合器輪替與動態排除機制**

實現負載分散的多聚合器輪替算法,每個聚合器僅負責R/N輪聚合任務,避免單點性能瓶頸。輪替選擇演算法在O(1)時間複雜度內完成聚合器選擇,當檢測到惡意行為時動態更新排除清單並重新分配任務,確保系統在部分節點故障或惡意時仍能持續運作。相較於BMA-FL的O(m²)聚合器協調開銷,本研究的輪替機制消除協調複雜度,實現真正的負載分散。

**4. 拜占庭容錯安全保證與理論分析**

證明在挑戰驗證模式下系統提供f < n/3的理論拜占庭容錯能力,要求至少2f+1個驗證者誠實參與共識。安全性分析顯示,在至少一個誠實驗證者積極監控的條件下,系統被攻破的機率趨近於零。相較於Krum在Non-IID環境下容錯度從f < n/2降至f < n/5,PBFT驗證提供理論保證的f < n/3容錯度且不受數據分佈影響,同時通過動態切換避免持續承擔O(n²)通訊開銷。

**5. 經濟激勵與質押機制整合**

整合質押與獎勵機制形成完整經濟安全模型,聚合器和驗證者需質押代幣參與系統,惡意行為導致質押損失,誠實參與獲得獎勵。質押門檻設計為參與獎勵的10-20倍,確保攻擊成本顯著高於誠實收益。經濟激勵與PBFT技術保證形成雙層防禦,即使經濟模型在極端攻擊收益情境下失效,PBFT驗證仍可檢測並阻止惡意聚合,提供defence-in-depth安全架構。

本研究的核心創新在於首次實現**威脅感知的動態共識切換**,填補了區塊鏈聯邦學習領域安全性與效率權衡的Pareto前沿空白,為實際部署提供兼具理論保證與實用效率的解決方案。

## 1.4 論文組織

本論文其餘章節組織如下:

**第二章 背景知識**:介紹聯邦學習基礎、區塊鏈與智能合約、拜占庭容錯與PBFT協議、Optimistic執行與挑戰機制、多聚合器架構與輪替演算法、經濟激勵與質押機制,為理解本研究的混合共識機制提供必要技術背景。

**第三章 相關研究**:批判性分析多聚合器架構方案(FLCoin、MCFLM-CB、BlockDFL、BMA-FL)、Optimistic共識方案(opML及其變體)、混合共識方法(EPP-BCFL、Layer 1 + Layer 2結合)、安全機制對比(Byzantine容錯聚合算法、密碼學方法),系統性總結七大研究缺口並明確本研究定位。

**第四章 研究方法與架構**:詳述系統架構設計、完整工作流程、多聚合器模型、挑戰機制運作、混合Optimistic-PBFT共識協議、排除與恢復機制、安全模型分析、計算複雜度比較,展示如何實現安全性與效率的動態平衡。

**第五章 實驗評估**:設計實驗驗證框架效能,包括正常運作效率測試、Byzantine攻擊防禦測試、可擴展性分析、與現有系統的比較實驗,量化動態切換機制的效率提升與安全保證。

**第六章 結論與未來工作**:總結研究貢獻,討論局限性與未來研究方向,包括與ZK-proof結合、支援更多聚合算法、大規模實際部署等展望。
